# Contributing to the Quantum Black Hole Simulation Project

We welcome contributions from the community to help us improve this project and advance the field of quantum computing. By contributing, you agree to abide by our Code of Conduct and to license your contributions under the Apache-2.0 License.

## How to Contribute

We welcome contributions in various forms, including but not limited to:

- **Bug Reports:** If you encounter a bug, please open an issue on our GitHub repository. Provide a clear and detailed description of the issue, including steps to reproduce it.
- **Feature Requests:** If you have an idea for a new feature, open an issue to discuss it. We are always open to new ideas and suggestions.
- **Code Contributions:** If you would like to contribute code, please follow these steps:
    1. Fork the repository and create a new branch for your feature or bug fix.
    2. Write clean, well-documented, and well-tested code.
    3. Ensure your code adheres to the project's coding style and conventions.
    4. Open a pull request with a clear and detailed description of your changes.
- **Documentation:** We welcome improvements to our documentation, including the README, docstrings, and other documentation files.

## Development Setup

To get started with development, you will need to set up a virtual environment and install the required dependencies. We recommend using `venv` for this purpose.

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Testing

We use `pytest` for testing. Before submitting a pull request, please ensure that all tests pass.

```bash
pytest
```

## Code Style

We follow the PEP 8 style guide for Python code. We recommend using a linter such as `flake8` or `pylint` to ensure your code adheres to this style.
